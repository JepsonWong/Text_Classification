基于Web的汽车维护咨询信息系统设计与实现
许铁　许钢
摘  要  针对当前汽车维修行业存在的问题，提出用计算机技术、Internet/Intranet技术，建立基于Web的汽车维护咨询信息系统，为提高汽车维护行业的整体效益服务。
关键词  Internet/Intranet  Web  汽车维护咨询信息系统  B/S模式
1  当前汽车维修行业存在的问题
　　①从汽车本身来看，汽车技术迅速发展，新的车型、新的功能越来越多，全球近15年来已有100多个品种，3,000多种车型问世，并且每个车种每年7月份即推出下一年的新车型。目前进入我国的已有600种之多，同时系统日益复杂。电控燃油喷射、电控点火装置、怠速控制、排放控制、进气控制、增压控制、警告提示、自我诊断、备用功能与失效保护、自动变速器ECT、防滑差速器ASD、加速防滑系统ASR、防抱死装置ABS、牵引力控制TRC、电控悬架TEMS、巡航控制、安全气囊SRS、雷达防撞、防盗装置、空调系统、自动座椅、中央门控、车载计算机、多功能行车记录和处理、自动导航及信息处理、电子地图、音响系统、通讯系统等。目前尚不敢说有什么人能完全通晓市上出售的，甚至是他所接触的一切汽车，更不要说这些汽车的技术细节。
　　②从维修业状况来看，我国汽车维修业有22万多家大小维修企业，从业人员大约有200多万人，而具有专业技术训练和较高文化基础的工程技术人员不足3%。维修人员文化素质较低，并且技术结构不甚合理，机电分家，综合排除故障的能力较差，常常由小故障修成大故障，局部故障修成综合故障。维修人员特别是专家级的维修人员奇缺，导致人们对汽车维修业工作失去信心，由于人员素质和技术发展的不相适应，使得汽车维修费用升高。
　　③从汽车消费者个人情况来看，正确地了解、使用新车的新功能，是减少故障的重要途径。而对绝大多数的消费者调查发现，约80%以上不懂得使用诸如：自动变速器O/D开关，换档模式，巡航控制等新功能。如买车后又要从头开始学习一些汽车的基本原理、构造和维修技巧，一方面没有这方面的教育，另一方面也感到驾驶中遇到的问题千差万别，无法预料。
　　④在国内方面随着改革开放的飞速发展，我国运输体系发生了重大变化。一是进口大量先进的汽车，二是国家从政策上也要求生产先进的汽车。在这种形势下，迫切地要求我们的维修行业，首先是转变观点，要从以磨损理论为依据的观念转变到更科学、更符合汽车运用实际的可靠性理论为依据的观念。要从定期解体检查为主转变到以不解体的外部检测诊断。
　　⑤由于语言方面上的原因，车辆维修的技术资料严重短缺和滞后，严重地影响了车辆维修质量。错判、错修的事件时常发生。
　　⑥检测仪器严重缺乏。由于现代汽车的复杂性大幅度提高，检测器几乎成为修车必备的工具，学习使用先进的仪器也成为我国汽车维修人员重要课题。
　　总之，汽车电子化的进程的飞速发展，引起了汽车性能划时代的发展。给汽车维修、汽车使用等诸方面带来前所未有的冲击，维修对象，经营模式，维修观念都发生了巨大的变化，其本质在于汽车维护观念的滞后，它既包括技术信息滞后，也包括管理信息滞后。
　　以上种种因素，从客观上就需要建立一种新的汽车维护咨询模式，它要及时地为维修人员提供各种车辆千差万别的技术信息，克服维修人员素质低，而造成的效率低，成本高的弊端。同时，可以为更广大的消费者提供高效的咨询服务。
　　以计算机技术为核心，将网络、数据库、专家系统、多媒体、语音热线服务及虚拟企业等先进的技术及管理方式有机结合，形成的综合咨询服务系统将有能力迎接这种挑战。
2  构建基于Web的汽车维护咨询信息系统
　　传统的企业网大多采用以Novell、Windows、Unix等网络操作系统为代表的局域网技术。它在网络内部具有较强的系统管理、传输能力，但在远程管理、远程信息共享、同外界的信息交换等方面有着明显的缺陷。而如今Internet已成为全球最成功的通信网络，它提供Email、FTP、WWW、News、Gopher、在线服务等诸多强大功能。尤其是WWW以全图形化的超文本链接方式，提供了寻找其它网络资源的方便、完整和快捷的解决方案。
　　特点：Intranet就是在传统的企业局域网基础上，采用Internet技术，以TCP/IP协议作为基础，以Web为核心应用，构成统一和便利的信息交换平台。基于Web计算机和网络技术的Intranet需要的启动资金少、建立与维护简单、用户使用方便。
　　基于Web平台的Intranet则大大扩充了信息处理和交换方式上的灵活性，将无论是内部的还是外部的、不同业务系统之间的信息集成为图文声并茂的多媒体信息，通过统一、方便的浏览以提供查询；信息流动是双向的。这样极大地扩展了信息资源开发、利用的深度与广度。并且Web平台利用CGI(通用网关接口)、JDBC(Java与数据库连接技术)、ISAPI(Internet服务应用程序接口)以及各数据库系统特有的工具等多种开发工具，建立HTML页面和企业数据库之间的连接，命名浏览器可以利用SQL语句实现查询数据库。
　　只有内部信息而无外部信息，是传统管理信息系统(MIS)的局限所在，打破这一限制，是Intranet迅速得到普及应用的重要原因之一。基于Intranet的企业管理信息系统采用一种全新的技术模式：Web Browser/Web Server(B/S)模式。B/S模式通常采用三层分布结构：浏览器―Web服务器―数据库服务器，如图1。

图1
　　B/S模式事实上是一种类似于终端/主机系统的结构模式，同时又具有客户/服务器模式的分布计算特性，其特点是集中管理。也就是说软件的程序、数据库以及其它一些组件都集中在服务器端，用户端除了浏览器(及所依赖的操作系统)之外无需其它软件和相关的管理维护工作。这样用户需要查询的资料和打印文档的数据都来自于同一个数据库，从而保证了数据的及时性与完整性。从管理的角度而言，程序代码的维护、更新以及数据库的备份和日常维护等都可在服务器端进行。

5  系统平台的选择
　　在Intranet上建立基于Web的信息咨询系统，有以下几个关键问题：
　　．WWW服务器平台的选择；
　　．数据库服务器的选择；
　　．WWW服务器与数据库服务器的互连；
　　．WWW应用的开发与调试。
　　WWW服务器平台。目前常用的WWW服务器平台的种类很多，例如：Microsoft公司的Internet Information Server (IIS)系列、Netscape公司的Enterprise Server、Fast Track Server系列、NCSA等。从技术方面分析，我们拟选择了Microsoft公司的IIS3.0作为汽车维护咨询信息系统WWW服务器平台。
　　(1)Internet站点的WWW服务器
　　Microsoft公司的IIS系列WWW服务器是Microsoft公司在Internet信息发布领域的主要软件，该软件虽然出现的时间比较晚，但是发展速度很快。目前，在很多领域中得到了广泛地应用。与其它公司的相关软件产品相比，Microsoft公司的IIS系列WWW服务器具有以下的优点：
　　．软件的安装、配置、管理简单，易于掌握；
　　．运行在Microsoft Windows NT操作系统平台上，具有很强的性能；
　　．兼容性好，支持标准的HTTP 1.0以及相关协议，支持CGI、WinCGI规范，可以开发基于WWW的应用；
　　．与Microsoft公司的其它相关软件产品，如Visual Studio97、SQL Server、Internet Explorer、Index Server、Exchange Server、Transaction Server等产品兼容性好，这些产品可以和IIS实现无缝集成，有助于开发更复杂、更高效的应用系统；
　　．Microsoft公司专门开发一组Internet应用程序接口(ISAPI)，提供了深入开发的手段；
　　．符合Microsoft公司的OLE/COM技术规范，可以采用ActiveX控件技术扩充基本服务器的功能；
　　．提供Internet Database Connection(IDC)、Active Server Pages(ASP)等高效开发工具，用于开发基于WWW的服务应用系统。
　　(2)数据库服务器
　　数据库系统选择Microsoft SQL Server6.5。Microsoft SQL Server 6.5是一个可伸缩的、高性能数据库管理系统。它专为分布式客户机/服务器计算环境而设计。它的内置数据复制功能、强大的管理工具和开放式的系统体系结构为发布信息提供了一个卓越的平台。它与Internet/Intranet紧密集成。SQL Server中Web Assistant自动以HTML格式发送信息，触发器驱动和预安排的任务很容易保持Web页的内容为最新。SQL Mail可以使得在SQL Server中通过Microsoft Exchange发送和接收电子邮件，或者来自SQL Server的数据填充Exchange公共文件夹。
　　(3)WWW服务器与数据库服务器的互连
　　汽车维护咨询信息系统Internet站点建设是以数据库为核心的Internet站点。因此为了实现汽车维护咨询信息系统Internet站点建设工程，尤其是各种基于Web的管理信息系统，必须实现WWW服务器与数据库服务器的互连，实现两者的互连主要有两种技术途径：
　　．采用适当的开发工具，实现互连；
　　．采用数据库服务器厂商提供的专门系统实现互连。
　　相比较而言，前一种方法的适应性更广，更灵活。而后一种方法在数据咨询方面的性能要优于前一种方法。由于我们采用了Microsoft公司的IIS作为WWW服务器，因此，不宜采用第二种方法实现WWW服务器与数据库服务器的互连。采用每一种方法实现互连基本上都是使用的一些WWW服务器扩充技术，使得WWW服务器可以执行一定的程序，由这些程序使用数据库接口与数据库服务器实现互连。
　　为了达到通用性，大部分厂商都是用ODBC作为数据库接口。ODBC是Microsoft公司推出的专门用于实现数据库互连的一种通用接口，该接口通过如图3的方法实行对不同数据库的访问。

图 3
　　Microsoft ODBC管理器实现ODBC的管理工作，主要是ODBC数据源的管理，所有使用ODBC的程序都是使用特定的数据源去访问数据库。数据库ODBC驱动接收程序的请求，并调用适当的通信程序与数据库引擎通信，数据库引擎中的数据库本地驱动执行程序的请求，并将结果返回给请求程序。
　　采用ODBC技术的最大优点是通用性好，用户程序并不会因为使用的数据库不同而作大量的修改，这样可以提高开发的效率和程序的可重用性，系统的升级改造也很简便。基于Web的汽车维护咨询信息系统采用ODBC作为访问数据库的工具。
　　SQL语言是访问数据库时使用的一种结构化查询语言，由IBM公司开发，是目前国际公认的数据库查询语言。SQL语言采用类似英语的语法结构，容易掌握，而且所有的数据库厂商都支持SQL语言。几乎所有的厂商为了扩充系统功能，都对SQL语言进行了不同程度的扩充，形成了一种简单的编程语言。在使用ODBC访问数据库时，为了系统的兼容性，我们只使用标准的SQL语言。
　　(4)WWW应用的开发与调试
　　WWW应用指的是用户使用普通的浏览器就可以完成所有的应用处理过程，而不需要在用户方安装特定的程序，这是传统的Client/Server处理模式的一个重大的改进。采用这种方式，可以大大减少开发人员的维护工作量，同时用户使用时的限制也大大减少。
　　在开发基于Web的信息系统中汽车维护咨询信息系统，我们采用Microsoft公司的Active Server Pages(ASP)作为开发工具。ASP是Microsoft公司在IIS3.0中推出的一种WWW应用开发工具，可以用来构造复杂的WWW应用。ASP使用了OLE/ISAPI接口，这个接口是Microsoft公司在开发Internet应用时的标准接口。
　　ASP除了提供一种程序手段之外，还提供了一组内建对象以及部件，这些对象与部件可以完成非常强大的功能，诸如数据库连接、文件访问等功能。同时，用户还可以自行设计新的部件以满足特定的要求。
　　ASP的内置对象有以下几种：
　　．Server对象：完成一些相关的需要WWW服务器处理的工作，例如生成部件实例、CGI编码等工作；
　　．Request对象：收集用户请求中的信息，以及用户的一些信息；
　　．Response对象：控制WWW服务器对用户请求的响应过程；
　　．Application对象：处理某一特定应用中需要所有用户共享的信息；
　　．Session对象：处理某一次会话过程中，用户的一些信息。
　　ASP的内置部件有以下几种：
　　．Ad Rotator：功能是实现一个简单的动画效果；
　　．Browser Capabilities：检查用户方浏览器的兼容性以决定程序的行为；
　　．Database Access：访问数据库；
　　．Content Linking：控制WWW连接的一种工具；
　　．File Access：访问服务器本地文件。
　　上面的对象和部件提供了一个很方便的开发环境，在基于Web的信息系统中主要是用数据库访问控件，由该控件使用ODBC与汽车维护咨询信息系统内部的后台数据库相连，完成数据库的操作工作。
6  结束语
　　采用国际上流行的Internet协议和Web技术，以WWW浏览的方式实现汽车维护咨询信息管理。这种技术是目前最新的技术手段之一，具有简洁、高效、使用方便等特点，并具有良好的可扩充性、可维护性等优点。因此，上述的综合咨询服务系统将会为提高汽车维护行业的整体效益提供帮助。
